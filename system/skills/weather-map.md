# 날씨 지도 (TV 뉴스 일기예보식) — kma_weather → Map

"전국 날씨 지도", "내일 전국 날씨", "주요 도시 날씨 지도" 류 요청이면 **TV 뉴스 일기예보처럼**
그린다. 각 도시 마커에 **이름 + 기온 + 날씨 이모지가 항상 보이게** 한다.

**흔한 실수 (하지 말 것):** 색 점만 찍고 온도를 색 범례로만 표시 → 지도엔 정체불명 점만 남는다.
Kakao 베이스맵은 국가 줌에서 광역시 이름만 띄우므로(강릉·청주·전주 등은 안 뜸) **베이스맵 라벨에
의존하면 안 된다.** 반드시 마커마다 `labelAlways: true` 로 우리 라벨을 그린다.

## 1. 데이터 — kma_weather `short` (단기예보), 도시별

각 도시에 대해 `kma_weather { action: "short", lat: <위도>, lon: <경도> }` → 시간별 예보 items.
- **오늘/내일 최고기온 = `TMX`, 최저 = `TMN`**(그 날짜의 값). 요청이 "내일"이면 내일 날짜 항목.
- **하늘상태 `SKY`**: 1=맑음 / 3=구름많음 / 4=흐림. **강수형태 `PTY`**: 0=없음 / 1=비 / 2=비·눈 / 3=눈 / 4=소나기.
- 도시가 많으면 병렬 호출. (short 는 lat/lon 주면 nx/ny 자동.)

## 2. 주요 도시 좌표 (검증된 중심 좌표 — geocoding 대체용, AI 기억 아님)

요청이 "전국"이면 위 목록 정도(권역 고루)를 쓴다. 특정 지역만이면 그 지역 도시들만.
목록에 없는 도시는 `kakao-map` geocoding 으로 좌표를 얻는다(기억으로 지어내지 말 것).

## 3. Map 렌더 — 도시별 마커에 항상 보이는 라벨

`render` 로 `map` 컴포넌트. 각 도시마다 마커 1개:

```
{
  "lat": 37.5665, "lon": 126.9780,
  "label": "서울\n☀️ 24~30°C",     // 도시명 + 날씨 이모지 + 최저~최고 (\n 으로 2줄)
  "labelAlways": true,               // ← 항상 보이는 텍스트 칩 (핵심). 없으면 hover 해야 보임
  "color": "red"                     // 최고기온 밴드 색 (보조 악센트)
}
```

- **`labelAlways: true` 필수** — 이게 없으면 점만 찍히고 이름·기온이 안 보인다.
- **날씨 이모지**(라벨 텍스트에): 맑음 ☀️ / 구름많음 ⛅ / 흐림 ☁️ / 비 🌧️ / 소나기 🌦️ / 눈 ❄️ / 비·눈 🌨️.
  (PTY 가 0 이 아니면 강수 이모지 우선, 0 이면 SKY 로.)
- **label 포맷**: `"도시명\n<이모지> <최저>~<최고>°C"` (예 `"강릉\n🌧️ 21~26°C"`). 요청이 최고기온만이면 `"서울\n☀️ 30°C"`.
- **color = 최고기온 밴드**(악센트, 보조): ≥33 `#b91c1c` / 30–32 `#ef4444` / 27–29 `#f97316` / 24–26 `#eab308` / ≤23 `#3b82f6`. 강수(비/눈)면 파랑 계열로 덮어도 됨.
- **legend**(선택): 온도 밴드 + 강수 표시. 단 색은 보조일 뿐 — **값은 라벨이 전달**한다.
- center/zoom 은 생략(마커 2+ 자동 fit). 전국이면 그대로 두면 한반도 전체로 맞춰진다.

## 4. 요약 텍스트

지도 아래에 한 줄 요약(오늘/내일, 전반 경향 — 예 "내일은 중부 흐리고 비, 남부 맑음, 서울 30°C").
표로 도배하지 말 것 — 지도가 본체.
